# SQL / Database Security Project

## Objective

Use PostgreSQL and SQL coursework to show database fundamentals and connect them to protecting sensitive data.

## Planned Topics

- Table design with primary keys and foreign keys
- Data types and constraints
- SELECT, WHERE, JOIN, and ORDER BY queries
- Basic data validation with CHECK, UNIQUE, and NOT NULL
- Notes about protecting sensitive information in databases

## Next Steps

1. Create a small example database scenario.
2. Document the tables, keys, and constraints.
3. Add example queries with short explanations.
4. Explain how good database design supports security and data integrity.
